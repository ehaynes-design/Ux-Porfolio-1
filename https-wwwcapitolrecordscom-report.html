Executive Summary

Overall, the Capitol records homepage passed all of the sections of the evaluation. There were a few errors that. I found after running the report.

  1. The page is missing a page title. This can. be fixed by adding a brief descriptive page title. A descriptive title helps users understand a page's purpose or content. Without a proper title, many users (especially those using screen readers or other assistive technology) may have difficulty orienting themselves to the page.

  2. The language of the document is not identified or a lang attribute value is invalid.The attribute is missing or is empty, or a lang attribute value is not a valid language identifier. Identifying the language of the page or page elements allows screen readers to read the content in the appropriate language. It also facilitates automatic translation of content.

  3. No heading structure, No

    -

    elements are present in the page. Provide a clear, consistent heading structure, generally one

    and sub-headings as appropriate. Except for very simple pages, most web pages should have a heading structure.

  4. No page regions or ARIA landmarks were found. No header, nav, main, footer, or aside HTML regions, or banner, navigation, main, or content info landmark roles were present in the page. Regions and ARIA landmarks identify significant page areas. Most web pages should have regions defined, particularly for the main content area.
